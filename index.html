How It Works
What the score measures
Every score request triggers a multi-stage pipeline. The system first checks for manipulation, then scores the wallet across multiple behavioral dimensions, and finally generates a confidence interval and actionable recommendation.
1
Integrity Screening
Before any scoring happens, the wallet's transaction graph is analyzed for Sybil patterns (fake identity clusters) and gaming behavior (attempts to artificially inflate a score). Multiple independent detection signals run in parallel. Wallets that fail integrity checks face score caps and penalties that cannot be overcome by volume alone.
Integrity
2
Behavioral Analysis
The wallet's on-chain activity is evaluated across multiple weighted dimensions. Each dimension answers a different question about the wallet's trustworthiness. Dimension weights are proprietary and reflect our research into which behavioral signals are most predictive of agent reliability.
Scoring
3
Integrity Adjustment
Raw dimension scores are adjusted based on integrity screening results. Flagged wallets receive caps and penalties applied to the relevant dimensions. This ensures that wallets engaged in manipulation cannot achieve high scores regardless of other activity.
Integrity
4
Composite Score
Adjusted dimension scores are combined into a single 0-100 composite using proprietary weighting. The composite is the headline number returned in every API response.
Score
5
Confidence & Recommendation
A confidence interval (0.0 to 1.0) is calculated based on the quantity and quality of available data. A recommendation is generated using priority-ordered logic that considers integrity flags, data sufficiency, and score thresholds. This is the output your agent should use to make decisions.
Output
Scoring Dimensions
Four questions about every wallet
Each dimension evaluates a different facet of on-chain behavior. Dimensions are independently scored and then combined using proprietary weights.

Reliability

"Does this agent follow through?"

Evaluates consistency and predictability of transaction behavior over time. Wallets with regular, sustained activity patterns across diverse counterparties are considered more reliable than those with erratic or concentrated behavior.

Viability

"Can this agent actually pay?"

Assesses financial health and sustainability. Considers the relationship between a wallet's balance, its transaction volume, and how its financial position changes over time. Wallets that maintain healthy, stable positions score higher.

Identity

"Is this a real, unique agent?"

Measures the wallet's uniqueness and organic legitimacy within the broader transaction network. Evaluates relationship diversity, network structure, and the absence of patterns associated with manufactured identities.

Capability

"Is this agent economically active?"

Gauges overall economic footprint and network influence. Considers transaction volume, activity breadth, and the wallet's role within the wider agent economy. Larger, more connected participants score higher.

Integrity Systems
How we detect manipulation
Two proprietary detection systems run before dimension scoring. If either flags a wallet, score caps and penalties apply automatically.
Sybil Detection

Catches fake identity clusters

Analyzes the wallet's transaction graph for patterns consistent with coordinated inauthentic behavior: circular fund flows, artificial relationship structures, and wallet creation patterns that suggest a single operator controlling multiple identities. Multiple independent signals are evaluated. Any confirmed pattern triggers score restrictions.

Gaming Detection

Catches score manipulation

Monitors for behavioral anomalies that suggest a wallet is trying to artificially inflate its score: sudden spikes in activity inconsistent with history, strategically timed balance changes, and patterns of activity that stop abruptly once a score threshold is reached. Each detection applies targeted penalties.

Confidence (0.0 to 1.0): How much data backs the score. A high score with high confidence is meaningful. A high score with low confidence means the wallet is under-observed and the score may shift significantly as more data arrives.

Recommendation: The actionable output. Possible values: flagged_for_review (integrity concern detected), insufficient_history (not enough data to score meaningfully), high_risk (low score), proceed (score and confidence support transacting), proceed_with_caution (mixed signals). The thresholds for each recommendation are proprietary and may be adjusted across model versions.
Transparency
Model versioning
Scores update as new data arrives. The system continuously indexes blockchain activity and periodically refreshes scored wallets. If a wallet's behavior changes, its score changes on the next refresh cycle.

The model itself evolves manually. Dimension weights, scoring thresholds, and integrity parameters are adjusted based on outcome data and ecosystem feedback. Every adjustment creates a new model version. Every API response includes a modelVersion field so you always know which version produced the score.

Outcome tracking is built in. The system monitors what happens after score lookups: did a transaction follow? Did a fraud report come in? This data informs model tuning decisions over time. Your usage makes the model better.

Integration
Add trust to your agent in one call
Score the recipient before every transaction. Gate payments on the recommendation field.
JavaScript
// Before sending USDC, check the recipient's reputation
const wallet = "0xRecipientAddress";
const { score, confidence, recommendation } = await (
  await fetch(`https://djd-agent-score.fly.dev/v1/score/basic?wallet=${wallet}`)
).json();

// Gate the transaction on the recommendation
if (recommendation === "proceed") {
  sendUSDC(wallet, amount);
} else if (recommendation === "proceed_with_caution") {
  sendUSDC(wallet, amount * 0.5);  // Reduce exposure
} else {
  log(`Blocked: ${recommendation}`);
}
