Scoring Dimensions
Five questions about every wallet
Each dimension evaluates a different facet of on-chain behavior, then they're combined into a single composite score. Model v2.1.0.

Reliability

"Does this agent follow through?"

Evaluates consistency and predictability of transaction behavior over time. Wallets with regular, sustained activity across diverse counterparties score higher than erratic or concentrated ones.

Like checking someone's track record — do they consistently show up and do good work?

Viability

"Can this agent actually pay?"

Assesses financial health and sustainability. Considers balance relative to transaction volume and how the wallet's position changes over time. Stable, healthy positions score higher.

Like checking if a contractor is financially stable enough to complete your project.

Identity

"Is this a real, unique agent?"

Measures the wallet's uniqueness and organic legitimacy. Evaluates relationship diversity, network structure, and the absence of patterns associated with manufactured identities.

Like verifying someone is who they say they are — not five fake profiles run by one person.

Behavior

"Does this agent act like a real one?"

Analyzes transaction timing patterns, consistency, and anomaly signals. Wallets that transact at regular intervals with organic timing patterns score higher than those with erratic bursts or suspiciously uniform activity.

Like watching how someone works day-to-day — genuine agents have natural rhythms, not robotic scripts.

Capability

"Is this agent economically active?"

Gauges x402 revenue earned (log-scaled), services operated, and the wallet's role within the wider agent economy. Agents that earn through x402 micropayments build verifiable economic track records.

Like checking how much business experience someone has — more connections and activity means more established.
Integrity Systems
How we catch fakes
Two proprietary detection systems run as part of the scoring pipeline. Flags from either system apply multiplicative penalties that stack — the integrity multiplier floors at 0.10 (90% max penalty).
Sybil Detection — "One person, many masks"

Catches fake identity clusters

Imagine discovering that five "different" freelancers you're considering all work from the same office, share the same bank account, and were all created last week. That's what Sybil detection catches on-chain: circular fund flows, artificial relationship structures, and wallet creation patterns that suggest one operator controlling multiple identities.

Gaming Detection — "Cramming before the test"

Catches score manipulation

Like someone padding their resume right before an interview — sudden spikes of fake activity designed to look legitimate. We monitor for behavioral anomalies: activity inconsistent with history, strategically timed balance changes, and patterns that stop abruptly once a score threshold is reached. Each detection applies targeted penalties.

The Output
What you get back
Every API response includes these key fields. Together they tell your agent everything it needs to make a decision.

Score

0 – 100

The headline number. A single composite of all five scoring dimensions, adjusted for integrity. Higher is better.

Tier

5 levels

A human-readable label derived from the score. Makes it easy to set thresholds without thinking about numbers.

Confidence

0.0 – 1.0

How much data backs the score. A high score with high confidence is meaningful. A high score with low confidence means the wallet is under-observed and may shift.

Recommendation

actionable output

The field your agent should use to make decisions. Uses priority-ordered logic combining integrity flags, data sufficiency, and score thresholds.

proceed proceed_with_caution insufficient_history high_risk flagged_for_review

Integration
Add trust to your agent in one call
Score the recipient before every transaction. Gate payments on the recommendation field.
JavaScript
// Before sending USDC, check the recipient's reputation
const wallet = "0xRecipientAddress";
const { score, confidence, recommendation } = await (
  await fetch(`https://djd-agent-score.fly.dev/v1/score/basic?wallet=${wallet}`)
).json();

// Gate the transaction on the recommendation
if (recommendation === "proceed") {
  sendUSDC(wallet, amount);
} else if (recommendation === "proceed_with_caution") {
  sendUSDC(wallet, amount * 0.5);  // Reduce exposure
} else {
  log(`Blocked: ${recommendation}`);
}
